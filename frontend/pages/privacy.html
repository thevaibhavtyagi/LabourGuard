Academic Proof of Concept

Engineering Trust in the Workplace.

// The Core Problem Statement

"Achieving Labour Laws compliance while actively reducing employer intrusion and surveillance."

This project was designed to prove that strict statutory compliance tracking does not require compromising the fundamental privacy rights of employees.

Our Engineering Approach

Instead of drafting a complex legal document, we believe in technical transparency. Here is exactly how we structured the database and logic to solve the problem statement.

The Minimum Viable Data

What we actually store in MongoDB.

  • Basic Identity

    Name and Work Email to generate unique session tokens and establish the HR relationship.

  • Cryptographic Credentials

    Passwords are never stored in plain text. We utilize bcrypt hashing for absolute database security.

  • Telemetry Timestamps

    Pure numerical ISO strings marking when an employee clicks 'Check-In' and 'Check-Out'.

The Engineering Boundary

What we explicitly refused to build.

  • No Geolocation Tracking

    We rely on cryptographic session trust. The system does not request or store GPS coordinates or IP tracking.

  • No Screen/Key Monitoring

    We track the duration of the work, not the nature of the work. No invasive background daemons are installed.

  • No Third-Party Data Selling

    Because this is a closed-loop academic project, data is restricted entirely to the Node.js/Mongo backend.

System Security & Stack

The underlying technologies driving the compliance engine.

To securely solve the problem statement, the system logic is separated into an isolated frontend and a secure REST API backend. The compliance engine runs server-side to prevent client-side manipulation of hours.
